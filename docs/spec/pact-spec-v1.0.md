# Pact
雇用・報酬・契約状態遷移インフラ仕様書 v1.0

## 1. プロダクト定義

Pact は、雇用継続・報酬増額・役割変更・契約終了（解雇）に至るまでの意思決定を、  
個人の評価や感情から切り離し、事前に定義された条件とデータによって状態遷移として確定させるための契約インフラである。

Pact の目的は、  
・昇給や解雇が恣意的・政治的に見えること  
・判断責任が特定個人に集中すること  
・突然の断絶によって怒りや暴力リスクが発生すること  
を構造的に防ぐことにある。

Pact は評価ツールではない。  
人を褒めず、裁かず、人格を扱わない。  
Pact が扱うのは「条件」「到達」「遷移」のみである。

## 2. 基本思想

Pact において、  
給料が上がることと、  
雇用が終了することは、  
同一のロジック上に存在する状態遷移である。

条件を上回れば、報酬レンジが上がる。  
条件を満たせば、役割が維持される。  
条件を下回り続ければ、契約が終了する。

この対称性を崩さないことを、設計上の最重要原則とする。

## 3. 絶対原則

以下は破ってはならない。

・人格評価を行わない  
・感情的表現を保存しない  
・基準を事後的に変更しない  
・昇給や解雇を裁量判断にしない  
・即日解雇を前提に設計しない  
・次の世界線を提示せずに契約を終了しない

判断主体は人ではなく、事前に定義された条件である。

## 4. 利用対象

・アメリカ企業の中間管理職  
・HR 部門  
・法務部門  
・報酬および雇用判断の説明責任を負うすべての立場

被雇用者本人は、自身に関するデータと最終レポートへの閲覧権を持つ。

## 5. システム構成概要

Pact は以下の三層で構成される。

### 5.1 Ledger Layer（実績ログ層）

役割ごとに定義された指標を、  
数値、期間、測定方法のみで継続的に記録する。

対象例。  
アウトプット量、品質指標、期限遵守率、コスト効率、役割固有 KPI 等。

自由記述は禁止、または定型文のみ許可される。  
態度、性格、協調性、意欲などの主観語は入力不可とする。

すべてのログは不可逆な時系列データとして保存される。

### 5.2 Threshold Engine（閾値・状態遷移エンジン）

各役割には、雇用開始時または役割変更時に以下を設定する。

・最低維持閾値（継続条件）  
・警告閾値  
・危機閾値  
・上位達成閾値（報酬増額条件）  
・状態評価期間

状態例。  
Growth（昇給候補）  
Stable（維持）  
Warning（警告）  
Critical（危機）  
Exit（契約終了）

状態遷移は、条件到達によって自動で行われる。  
マネージャーは遷移を決定しない。

### 5.3 Decision Output Layer（決定出力層）

状態遷移に応じて、以下を自動生成する。

・Salary Adjustment Notice（報酬改定通知）  
・Role Continuation Notice（役割維持通知）  
・Pact Report（契約終了レポート）

すべての出力は同一フォーマット、同一論理で生成される。

## 6. 報酬増額仕様

報酬増額は「評価」ではなく、状態遷移として扱われる。

上位達成閾値を一定期間継続達成した場合、  
または費用対効果が想定レンジを超えた場合、  
自動的に報酬レンジが切り替わる。

通知文には以下を含める。

・達成した指標と期間  
・適用された新報酬レンジ  
・次の上位レンジ条件

人格的称賛や感情的表現は禁止される。

## 7. 契約終了（解雇）仕様

最低維持閾値を下回る状態が、定義された期間継続した場合、  
契約終了状態へ遷移する。

遷移時には必ず Pact Report を生成する。

## 8. Pact Report 仕様

Pact Report は、安全確保と未来提示を目的とした公式文書である。

必須記載項目。

・役割定義と評価期間  
・継続雇用が成立していた条件  
・費用対効果がマイナスに転じた時点  
・未達だった指標と閾値  
・改善によって継続可能だった条件  
・他環境で強みとなり得る要素  
・再就職時に利用可能な要約

禁止事項。

・人格評価  
・断定的非難  
・感情的表現  
・主語が「あなた」の文章

## 9. UX / 運用原則

・報酬改定、警告、契約終了はすべて同一導線で通知する  
・口頭説明のみは禁止する  
・密室一対一での解雇通知を推奨しない  
・通知と同時に必ず文書を提供する

目的は納得させることではない。  
世界が連続していると理解させることである。

## 10. セキュリティ・法務前提

・全ログは改ざん不可  
・役割ベースアクセス制御  
・監査ログ保持  
・被雇用者本人の閲覧権保証

Pact は訴訟を防ぐことを保証しない。  
ただし、  
説明不能な昇給、  
説明不能な解雇、  
を構造的に排除する。

## 11. プロダクトの位置づけ

Pact は、  
人を切るための道具ではない。  
人を評価する道具でもない。

給料を安全に上げるため。  
契約を安全に終わらせるため。  
怒りと責任が個人に集中しないため。

組織内の重力を安定させるための、  
雇用と報酬の契約インフラである。
