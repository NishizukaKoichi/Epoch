# koichinishizuka.com UI仕様書
Version 1.0
Status: Final / Implementation-Complete Spec

## 1. 目的

### 1.1 このサイトの役割

・作者としての「仕様ライブラリ棚」を提示すること  
・各プロダクトの固定仕様へ、最短で到達できる導線を提供すること  
・商用サイトのような売り込みや勧誘と誤認されない情報設計にすること  
・思想・価値観と、実装・仕様を明確に分離し、第三者が参照しやすい状態を保つこと

### 1.2 このサイトが解決しないこと

・課金、決済、購入導線の提供  
・プロダクトのオンボーディングやアカウント作成  
・機能説明を目的としたマーケティング最適化  
・SEO 集客を目的とした量産記事運用

## 2. 主要コンセプト

### 2.1 “Library, not Store”

・閲覧者が「買う」より先に「理解する」ための棚として扱う  
・CTA は設置しない、もしくは仕様参照のための最小限リンクに限定する

### 2.2 “Three artifacts”

・Sigil  
・Epoch  
・Talisman

この3つを中心に据え、  
Pact は強い現実領域（雇用・報酬・契約状態遷移）を扱うため、  
周辺配置として扱い、仕様参照の導線は維持しつつ誤読コストを抑える

## 3. 情報設計（IA）

### 3.1 グローバル構造

・Home  
・Library  
・Notes  
・About  
・Contact

### 3.2 優先順位

・最優先は Library への到達  
・次点は各 Library Detail から固定仕様（Spec）への到達  
・Notes は補助情報であり、Library の役割を侵食しない

## 4. グローバルナビゲーション仕様

### 4.1 表示位置

・ヘッダーに常時表示  
・スクロール時の縮小は許可するが、非表示にはしない

### 4.2 ナビ項目

・Library  
・Notes  
・About  
・Contact

### 4.3 ロゴ／サイト名挙動

・クリックで Home に遷移  
・Home は「入口」として扱い、情報を詰め込みすぎない

## 5. ページ仕様

### 5.1 Home

#### 5.1.1 目的

・このサイトが「作者による仕様ライブラリ」であることを即座に理解させる  
・Library への導線を最短にする

#### 5.1.2 コンテンツ構造

・Hero  
　・1行の定義文  
　・補足1行  
　・主要リンクは Library のみ

・Featured Libraries（3カード固定）  
　・Sigil  
　・Epoch  
　・Talisman  
　各カードは  
　「定義1行」  
　「扱うもの／扱わないもの 各1行」  
　「Spec へのリンク」  
　のみを表示する

・Recent Notes（任意、最大3件）  
　・Notes へのリンクのみ

・Footer

### 5.2 Library（一覧）

#### 5.2.1 目的

・全プロダクト仕様の棚として機能させる  
・閲覧者が比較可能な最小情報のみを揃える

#### 5.2.2 表示形式

・カード型一覧  
・並び順は固定

　・Sigil  
　・Epoch  
　・Talisman  
　・Pact  
　・その他が存在する場合は下段に隔離

#### 5.2.3 カード内容（必須）

・名称  
・一行定義  
・扱うもの（3語以内を推奨）  
・扱わないもの（3語以内を推奨）  
・リンク  
　・Spec  
　・Value（存在する場合のみ）  
　・MVP（存在する場合のみ）

### 5.3 Library Detail（各プロダクト詳細）

#### 5.3.1 目的

・固定仕様へ到達するための中継ページ  
・説明を増やしすぎず、仕様本文を読ませる

#### 5.3.2 セクション構造（固定）

・Definition（1段落）  
・Non-Goals（扱わないこと、短文）  
・Artifacts（リンク集）  
　・Spec  
　・Value  
　・MVP  
　・Flows  
　・Done  
　存在しないものは表示しない

・Integration stance  
　・他プロダクトとの関係を1段落で示す

・Status（Draft / Final / Deprecated）

#### 5.3.3 禁止事項

・長いマーケティングコピー  
・価格の提示  
・比較煽り  
・購買を目的とした CTA

### 5.4 Spec Viewer（仕様本文ページ）

#### 5.4.1 目的

・第三者が参照しやすい表示  
・全文コピー可能  
・章単位での参照が安定すること

#### 5.4.2 UI要件

・デスクトップでは左に目次、右に本文  
・モバイルでは目次は折りたたみ  
・各見出しにアンカー ID を付与  
・見出し単位で URL 共有が可能  
・本文は可読性を優先し、等幅にはしない  
・コードブロックのみ等幅、横スクロールを許可

#### 5.4.3 文書スタイル

・番号付き見出し＋改行のみを基本とする  
・区切り線や過剰装飾は行わない  
・箇条書きは「・」を保持する

### 5.5 Notes（一覧）

#### 5.5.1 目的

・思考メモや周辺文章の置き場  
・Library を補助するが、主役にはならない

#### 5.5.2 仕様

・時系列一覧  
・タグは任意  
・Notes 内でのプロダクトリンクは許可  
・Notes が Spec を置き換える構造は禁止

### 5.6 Note Detail

・本文表示  
・前後記事ナビ  
・関連リンク（最大3）

### 5.7 About

・作者の簡潔な紹介  
・設計思想の短い宣言  
・外部リンクは最小限  
・Contact への導線を明示

### 5.8 Contact

・連絡方法を1つに統一  
　・メール、もしくはフォーム  
・SNS の埋め込みは原則行わない

## 6. 共通コンポーネント仕様

### 6.1 Header

・左：サイト名  
・右：グローバルナビ  
・モバイルではハンバーガーメニューを許可

### 6.2 Footer

・著作表示  
・簡易リンク  
・最終更新日（任意）

### 6.3 Library Card

・タイトル  
・一行定義  
・扱う／扱わない  
・Spec リンクを最優先表示

### 6.4 Status Badge

・Draft  
・Final  
・Deprecated（必要時のみ）

### 6.5 Link policy

・外部リンクは別タブ  
・内部リンクは同一タブ  
・Spec リンクは常に視認性を高くする

## 7. デザイン仕様

### 7.1 トーン

・静的  
・高コントラスト  
・余白を多く取る  
・情報密度は低く、深さはリンク先に委ねる

### 7.2 タイポグラフィ

・本文は可読性重視のサンセリフ  
・見出しは太さのみで差分をつける  
・行間は広め

### 7.3 色

・アクセントカラーは1色のみ  
・Status Badge は色分けを許可  
・背景はライトまたはダークのどちらかに固定

## 8. レスポンシブ仕様

### 8.1 ブレイクポイント

・モバイル  
・タブレット  
・デスクトップ

### 8.2 ルール

・Library 一覧はモバイル1列、デスクトップ2〜3列  
・Spec Viewer はモバイルで目次を折りたたむ  
・スクロール中も「目次へ戻る」導線を用意する

## 9. アクセシビリティ

・キーボード操作で全導線に到達可能  
・フォーカスリングを消さない  
・画像には代替テキストを付与  
・見出し階層の破綻を禁止  
・十分なコントラスト比を確保する

## 10. パフォーマンス要件

・初期表示は軽量  
・画像は遅延読み込み  
・フォントは最小構成  
・Spec ページは静的配信を優先

## 11. SEO・共有仕様

・各ページに title と description を設定  
・OGP（タイトル、概要、画像）を設定  
・Spec は見出し単位で共有可能  
・canonical を設定

## 12. 計測・観測

### 12.1 目的

・Library への到達率  
・Spec の閲覧深度  
・Notes が過剰に消費されていないか

### 12.2 イベント（最小）

・view_home  
・view_library  
・open_library_detail  
・open_spec  
・spec_anchor_copy  
・view_notes  
・open_note

### 12.3 禁止

・読者の評価指標化  
・滞在時間による最適化の強制

## 13. 運用仕様

### 13.1 更新方針

・Spec はバージョンを明示し、差分は別ページで管理する  
・Final と明記された Spec は原則変更しない  
・変更が必要な場合は新バージョンを発行する

### 13.2 壊さないルール

・Library の並び順を頻繁に変えない  
・Home の導線は常に Library 最優先とする  
・Notes は Library を置き換えない

## 14. Done 定義

以下を満たした時点で koichinishizuka.com UI は完成とする

・Home から Library へ1クリックで到達できる  
・Library から各 Spec へ2クリック以内で到達できる  
・Spec Viewer が見出しアンカー共有に対応している  
・モバイルで目次が実用的に機能している  
・アクセシビリティ要件が満たされている  
・計測イベントが最小構成で実装されている  
・購買 CTA が存在しない、もしくは参照リンクに限定されている
