# **Sigil**
Environment Pre-Learning Specification Platform

【1】プロダクト定義

**Sigil** は、会社・チーム・店舗・プロジェクトといった場に適用される  
前提条件・判断基準・完了条件を、**術式として文章で構築・固定**し、  
参加前に誰でも参照できる状態で公開・共有するための SaaS である。

本プロダクトが扱うのは、人ではない。  
教育でも、育成でも、評価でもない。

扱うのは、その場に適用される  
「どのような術式が敷かれているか」という**仕様そのもの**である。

**Sigil** の目的は以下に集約される。

・場に作用する前提条件を、術式として事前に明示すること

・参加予定者が、術式の内容を事前に把握できる状態を作ること

・学習・確認が行われなかった場合、それを能力や態度ではなく、  
　**術式理解を優先しなかったという選択の結果**として扱える状態を成立させること

説明責任は、術式が公開されていることをもって完了する。

【2】想定ユーザー

【2.1】術式構築者（Primary User）

・会社・部署・店舗・プロジェクトにおいて、  
　**適用される術式の設計責任を持つ立場**

・判断基準や完了条件を、空気や慣習、口頭説明に依存させたくない人

・人の理解力や善意ではなく、  
　**術式として記述された仕様そのものが効力を持つ状態**を作りたい人

**Sigil** 上で行うのは、説得ではない。  
教育でもない。

発動条件・効果範囲・制約を持つ術式を、  
文章として構築し、固定し、公開することである。

【2.2】術式適用圏への参加予定者（Secondary User）

・入社予定者

・配属・参画予定者

・業務委託・外部協力者

このユーザーは学習者ではない。  
**術式の内容を読んだ上で、その効力圏に入るか否かを判断する主体**として扱われる。

【3】基本原則

**Sigil** は以下の原則を持つ。

・人の内面、動機、価値観を扱わない

・判断はすべて「術式への適合／不適合」として行う

・読める状態で公開されていることをもって、術式は発動前開示されたものとみなす

・読まなかったことは理解不足ではなく、  
　**術式理解を優先しなかったという選択**として扱う

感情的配慮、補足説明、口頭フォローは、術式の外に置かれる。

【4】利用フロー

【4.1】術式構築者の利用

Sigil 上に、自身の管理する場を作成する。

その場に適用される術式を、文章として登録する。

発行された術式 URL を、新人・候補者に共有する。

「事前に内容を確認してください」と伝える。

これ以外の説明義務は発生しない。

【4.2】術式適用圏への参加予定者の利用

共有された URL にアクセスする。

公開されている目次を確認する。

必要な章を事前に読む。

自身がこの術式の効力圏に入るかどうかを判断する。

理解・記憶・参照は、すべて本人の責任として扱われる。

【5】UI仕様

【5.1】公開エントリページ

最初に表示される情報は以下に限定される。

・術式が適用される場の目的

・想定される判断の種類

・完了条件（Done）の厳しさ

・術式が効力を持つ責任範囲

操作可能なのは「目次を見る」のみとする。  
ログインや登録は不要とする。

【5.2】目次 UI（中核）

UI モデルは書籍の目次とする。

・全章を一画面で一覧表示

・未読状態が視覚的に判別可能

・章の順序は固定

標準章構成は以下とする。

術式の前提条件  
発動条件  
日常的に発生する判断  
完了条件（Done 定義）  
権限と責任の境界  
例外・逸脱時の扱い

目次はナビゲーションではない。  
術式全体の構造を一望させるものとして機能する。

【5.3】仕様本文ページ

・装飾は行わない

・文章・定義・条件のみを表示する

常に以下を表示する。

・現在の章

・適用範囲

操作は以下に限定される。

・目次に戻る

・次の章へ進む

【6】外部術式・他環境ルールの参照と差分確認

**Sigil** は、他の場で使用されている術式やルールを  
比較対象として参照できる機能を持つ。  
目的は再利用ではない。

・他術式の前提条件を確認する

・自術式との差分を明確化する

・自術式で言語化されていない部分を特定する

制約は以下とする。

・既存術式の自動上書きは不可

・紐づけのみ可能

・主語は常に自術式

【7】判定ロジック

以下の状態が確認された場合、

・術式の主要前提を把握していない

・完了条件（Done）を誤認している

・判断基準を参照できない

その人は、  
**術式理解を優先しなかった＝別の目的を優先した**  
と判定される。

これは能力評価ではない。  
術式不適合による結果として扱われる。  
不採用・非参画は、合理的判断として成立する。

【8】料金体系

課金軸は人数ではなく、  
**術式仕様の固定レベル**とする。

Free（Public）

・公開術式の作成

・目次・仕様の閲覧

・外部術式参照

Private Spec

・非公開術式

・新人・候補者への共有

・参照履歴（閲覧有無）

Fixed Spec

・術式変更履歴

・外部術式との差分管理

・術式ロック

Enterprise

・削除不可

・監査用閲覧

・契約・制度用途

【9】成功状態（Done 定義）

**Sigil** が正しく機能している状態は以下。

・参加者が術式前提を理解した上で参画する

・初期質問が具体的かつ限定的になる

・「聞いていない」が発生しない

・術式と合わない人が事前に離脱する

日常的な運用や感情的フォローは必要とされない。

【10】提供価値の整理

**Sigil** が提供するのは、教育ではない。  
育成でもない。  
評価でもない。

提供するのは、  
**術式として固定された前提条件の事前開示**と、  
**それを理解した上での合理的な参加判定**のみである。
