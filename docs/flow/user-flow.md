# User Flow Definition

必須項目: 以下の REQUIRED プレースホルダを埋めること。
未記入のまま次工程へ進まない。

## entry_channels
テンプレ配布ページ/紹介/コミュニティ/検索/既存プロダクトの社内共有。
（どこから入ってくるか：例 X/紹介/検索/直URL/招待 など）

## first_success
サンプルのplanKeyでCheckoutが起動し、Webhookでentitlementが付与されることを確認。
（初回の“成功体験”は何か：最初に価値が確定する瞬間）

## payment_consideration
自分の仕様に置き換えた時に、今週中に有料検証を始められると判断した時。
（課金検討に入る条件：何を見て「払うか」を考え始めるか）

## post_purchase
新しい仕様や検証仮説が出たタイミングで再度テンプレを使って実験を回すために戻る。
（支払い後の再訪理由：翌日/翌週に何のために戻ってくるか）
