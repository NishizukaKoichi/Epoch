# Value Definition

必須項目: 以下の REQUIRED プレースホルダを埋めること。
未記入のまま次工程へ進まない。

## who
仕様書は書けるが、課金・権限・計測の初期実装に時間を取られている小規模プロダクトチーム。
（誰が）

## situation
MVPを短期間で有料検証したいが、決済/権限/計測の土台が未整備で開発が止まっている。
（どの状況で）

## pain
価値検証よりも課金・Webhook・権限更新の実装に工数が吸われ、リリースが遅れる。
（何に困っていて）

## after_paying
テンプレの導入だけで決済→Webhook→権限付与の流れが動くと確定し、仕様に集中できる。
（支払った瞬間に、どんな不確実性が消えるのか / 何が確定するのか）
